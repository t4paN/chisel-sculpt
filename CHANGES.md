# Chisel — Changes

Short, chronological log of notable changes. Newest on top.

## 2026-06-07 — Paint mode: per-vertex albedo brush (vpaint core)

- **New Paint brush** lays per-vertex colour onto the model — the 0.1.4 north-star item. Reachable from the toolbar Paint button (Q/E to cycle); an ImGui colour swatch sets the brush albedo. GPU dab kernel, mirror-aware, full undo/redo, multi-entity correct.
- **How it works:** vpaint is the mask system with packed RGBA8 instead of a scalar. Storage is `mesh.color` (`vector<uint32_t>`, `r|g<<8|b<<16|a<<24`, `0xFFFFFFFF`=white=unpainted → renders byte-identical to an unpainted model). Rendered as a `GL_UNSIGNED_BYTE` normalized vertex attribute (loc 4) wired on **both** the working VAO and per-entity display VAOs; matcap frag multiplies shade by `vColor.rgb`. The compute kernel (`compute_color.cpp`, binding 28) **lerps** old→brush colour (`mix`, not mask's additive accumulate) so dabs converge to the target; erase lerps toward white. Pen-up reads the dirty colours back into `mesh.color` for a `Kind::PAINT` undo entry (per-entity, level-agnostic).
- **Known gaps (next):** paint does **not** yet survive subdivide / remesh (silently dropped), and is not written to `.chisel` save or OBJ export. Persistence is the next chunk.

## 2026-06-07 — SDF: optional mirror-symmetric voxel-merge (Y / M / N)

- **The voxel-merge confirm dialog now offers a mirror-symmetric merge.** `Y` = faithful merge (the existing, possibly-asymmetric union — correct for a pure print export); `M` = mirror-merge (symmetric about the app's mirror plane x=0, mirror-editable); `N`/`ESC` cancel; `[`/`]` still nudge resolution. Mirror is **not** the default — a join-for-print may legitimately need asymmetry.
- **Why M exists:** marching cubes bakes a *fixed* grid-space diagonal per ambiguous face, blind to x=0, so a +x feature and its −x twin tessellate the same world direction. Geometry can be symmetric while topology is not, which made `build_mirror_x_map`'s partner pairing approximate → mirror-aware smooth fought itself → seam artifacts on the mirrored side. The merged mesh was printable but not mirror-stack compatible.
- **How M works (self-contained in `sdf.cpp`, never via `perform_remesh`):** the x grid window is forced symmetric `[-X,+X]` and `origin.x` is snapped so a corner layer lands exactly on x=0 — so no cell straddles the plane and every +x tri has all verts at x≥0 (seam verts exactly 0). Then: `clip_to_plus_x` drops −x tris + planar seam sails and compacts; `relax_to_field` runs on the +x half with the seam pinned (new optional pin mask) so the seam loop can't drift off-plane; `reflect_across_x` appends X-negated copies of non-seam verts (seam verts shared) with swapped-winding tris. Symmetric by construction → exact partner map. `main.cpp` sets `mirror_topology(true)` for an M-merge so `refresh_mirror_map` builds the exact (not spatial) map. The `Y` path is unchanged.
- **Killed the seam sliver column on the mirror merge.** A corner-aligned grid puts the first +x vertex on a seam→interior edge at `x=voxel·t`; when the iso-crossing sits near the seam corner (`t→0`) that vert is ~coincident with its seam vert → a column of near-zero-width tris down x=0. Two passes on the +x half before reflecting (so symmetry stays exact): **(B) `weld_near_seam`** welds each +x vert within `0.30·voxel` of the seam onto its closest seam vert across a short edge (≤`0.50·voxel`), drops the collapsed tris, compacts — removes the slivers at the source; **(A)** the seam is no longer rigidly pinned during relax — each seam vert now averages only its two seam-neighbours, stays exactly on x=0, and reprojects onto the surface via the x-stripped (in-plane) gradient, so the seam loop evens out and the first interior row isn't pulled thin. User-confirmed clean; mirror merge called production-ready.

## 2026-06-06 — SDF: relax the merged triangulation onto the iso-surface

- **Voxel-merge output is now evened out without losing shape.** Marching cubes lands vertices on grid edges → blocky, axis-aligned tris with cell-corner slivers ("little triangles hiding inside the squares"). After weld, a tangential relaxation pass (8 iters) now reads the signed field back and, per vertex, moves toward the 1-ring centroid *in the tangent plane only* (normal stripped via the SDF gradient), then reprojects onto the field's zero level set. Tris spread to uniform spacing while the silhouette is held exactly — unlike the smooth brush, which deflates the form because it moves vertices off-surface. Pure vertex relaxation (topology/count unchanged), self-contained in `sdf.cpp` (never routed through `perform_remesh`). Second readback, but one-shot/user-paced like the soup readback — the no-readback rule is stroke-only.

## 2026-06-06 — Fix: edit-mode crash after merge/remesh on a single entity

- **`active_mesh(): Assertion 'e && "no active entity"'` abort fixed** — `refresh_for_edit_mode` called `set_active(0)` in the `alive_count() <= 1` branch, but entity ids start at 1 (`next_id_` inits to 1), so id 0 means "no active entity": that stranded `active_id_` and the next `active_mesh()` aborted. Latent for a lone original mesh (rarely re-entered via SELECT), but a **merge** (collapses the selection to one entity with a nonzero id) → remesh → switch to EDIT hit it every time. The pre-crash frame where "only the mirrored half renders" is the same `active_id_ = 0` state — the render loop has no active entity to draw from the working VAO. Now `refresh_for_edit_mode` resolves to a live entity (current active if it survived, else the first alive one) instead of zeroing.
- **Remesh now collapses a stale multi-selection** — remesh only rebuilds the active entity, so a leftover multi-selection (and its deselected tint) was stale afterward; `collapse_selection_to_active()` drops it back to the remeshed entity.

## 2026-06-06 — Remesh: fix NaN-shredding bug, tighten the band, thin the seam

- **Fixed a NaN that silently destroyed meshes on remesh** — the GPU tangential-smooth shader did `normalize()` on the raw vertex normal. A momentarily-degenerate (zero-length) normal — far more common under heavy split/collapse load — makes that `0/0 = NaN`, which leaks into `out_pos.y/z` (the `x` slot is masked by the seam `sign()/max()` clamp, so it stays finite). A single NaN coordinate poisons `build_mirror_spatial`'s mean-edge sum → `tol=NaN` → 0 paired vertices → the whole mirror is unpaired and the model shreds. Now guard the normalize: a zero-length normal has no tangent plane, so leave the vertex put (`compute_remesh.cpp`). Fingerprint was `[mirror] spatial rebuild: 0 paired … tol=-nan, edge=-nan` with finite, balanced +x/-x.
- **Tightened the isotropic accept band toward canonical** — split now fires above `1.4·target` (was `high·1.1 = 1.54·target`) and collapse below `0.8·target` (was `0.6·target`), narrowing the "acceptable" edge-length window from 2.57:1 to the Botsch–Kobbelt 1.667:1. Hits both residuals at once: fewer oversized tris (split reaches them) and fewer ultra-fine tris (collapse clears them). Stable — `high < 2·low` holds, so no split↔collapse oscillation. Mask path, seam-border collapse guard, and pinned/seam handling unchanged.
- **Let the mirror seam decimate to target spacing** — seam verts are pinned for the whole split/collapse/smooth loop, so the center line stayed frozen at *input* density while the interior re-spaced to target, leaving a visibly finer band along x=0. The collapse boundary-edge guard treated every seam edge as a protected border (a seam pin + interior vert looked identical to a mask border), so seam edges never collapsed. Now the guard counts only **off-seam** pins (`|x| ≥ seam_tol`) as border pins, so a pure seam edge `[seam, seam, interior]` is free to collapse pinned-pinned (survivor snapped exactly to x=0), while a true mask/selection border `[off-seam-pin, interior, interior]` is still blocked. Seam thins to match the interior; center line stays watertight and symmetric.

## 2026-06-05 — Remesh: don't destroy unselected meshes

- **Remesh now edits only the active entity, leaving every other mesh intact** — like the save pipeline, the remesh trigger predated multimesh and ended with `scene.reset_to_single_mesh(0)`, which tore down every non-active entity (a bust's separate eyes vanished on remesh). But `perform_remesh(*mesh, *multires)` already rebuilds the **active** entity's mesh + multires base entirely in place (adjacency, normals, mirror map, fresh level-0 stack). The scene collapse was pure collateral. Dropped the `reset_to_single_mesh` call; the one bit it still did that we need — resetting the active entity's `subdiv_level` to 0 — is now an explicit `scene.active_entity().subdiv_level = 0`. `refresh_mirror_map()` + `sync()` both scope to the active entity (inactive display VAOs only re-upload when dirty), so unselected meshes keep their geometry, multires, and undo history untouched. Active entity's undo is still cleared (its topology changed).

## 2026-06-05 — Session save: persist the whole multimesh scene

- **`.chisel` now saves every entity, not just the active mesh** — the save pipeline predated the multimesh refactor and serialized only `*mesh`/`*multires` (the active entity), collapsing a multi-mesh scene (e.g. a bust + separate eyes) to one mesh on save *and* on load (`reset_to_single_mesh`). New format **VERSION 2**: global `CAMR`/`OPTS` + a `SCEN` chunk (`active_id`, `next_id`, `mirror_use_topology`, `selected_ids`) + one `ENTY` chunk per committed entity, each carrying `{id, subdiv_level, mesh body, multires body}`. Mesh/multires byte layout is unchanged — the old chunk writers were just factored into header-less `write_mesh_body`/`write_multires_body` reused per entity. Save iterates `scene.entities()`, skipping `preview` (uncommitted INSERT) entities.
- **Load rebuilds the scene via `Scene::load_entities`** — tears down all current entities + their display VAOs, recreates each record with its saved id, cascades any locked multires stack up to `current_level` (preserving the saved mask, matching the legacy single-mesh load), rebuilds adjacency/normals/mirror per entity, then restores active + selection (sanitized against loaded ids) and bumps `next_id` past every loaded id so later inserts can't collide. Per-entity undo histories start empty (undo is intentionally not serialized). Caller does `refresh_mirror_map` + `sync()` after, exactly like `reset_to_single_mesh`. Result: a saved bust-with-eyes reloads with all meshes, multires, camera, and the *same* active/selected mesh it had at save time.
- **v1 files still load, in a removable module** — `project_file_v1.{h,cpp}` is fully self-contained (own chunk parser, no shared helpers) and rehydrates a pre-v2 single-mesh file into a one-entity `ProjectData` (id 1, selection `{1}`, `next_id` 2, mirror-topology = the stack's locked state — the old behaviour). `load_project` dispatches on version: `1` → `load_project_v1`, else v2. Dropping legacy support later is a 3-line checklist (delete the two files, the dispatch branch, the CMake entry), documented in the header. New struct `EntityRecord` (`entity_record.h`) is the GPU-free `{id, subdiv_level, mesh, multires}` shared by the file layer and the Scene rebuild so neither pulls in OpenGL.

## 2026-06-02 — Remesh: pre-mirror seam consolidation (kill clip slivers)

- **Tiny triangles at the x mirror plane** — replaced the post-mirror `cleanup_seam_slivers` with `consolidate_seam`, run on the +x-only half (after the −x drop + compact, *before* reflection) inside `mirror_positive_half`. Root cause: plane-clipping an arbitrary triangulation against x=0 mints a near-zero seam edge wherever a pre-existing vertex sits just off the plane (the cut point on its neighbor edge lands ≈ on top of it) — those are the slivers hugging the centerline. New pass snaps any +x vert with `seam_tol ≤ |x| ≤ 0.45·edge` that has a true seam neighbour joined by a short edge (`< 0.6·edge`) onto x=0; the **edge-length gate is the discriminator** — clip slivers carry a near-zero seam edge, while a legit seam-adjacent apex on a surface crossing the plane ~perpendicularly sits ~0.87·edge out with ~0.87·edge seam edges and is excluded. Then welds coincident seam verts (`(y,z)² < (0.0625·edge)²`), drops degenerate + newly-all-seam sail tris, compacts; fully-masked tris preserved. Operating on the +x half *before* reflection makes the seam a clean single edge-loop, so the mirror is symmetric by construction — no `mirror_x_map` twin-chasing — and it reaches the full `[0.1, 0.5]·edge` band the old 0.1·edge cleanup couldn't. `mirror_positive_half` gained a `target_edge` parameter. Console line: `[seam-consolidate] snapped N, welded M, culled K`. GUI-verified: seam slivers down noticeably (a few survive — the upstream brush-side seam clamp is the remaining structural fix).

## 2026-06-01 — Pen pressure sensitivity (X11)

- **Graphics-tablet pressure** — new `src/tablet.cpp` / `include/tablet.h`: reads the stylus `Abs Pressure` valuator straight off **XInput2** (the same layer Krita/Blender use; GLFW has no tablet support). Own X display connection, raw motion selected on root (`XIAllMasterDevices`) + hierarchy events (`XIAllDevices`, separate mask — mixing the two is a `BadValue`), pressure decoded from `raw_values` and normalized by the device's reported range. `XIHierarchyChanged` triggers a device rescan, so plugging the tablet in after launch works. Stylus + eraser both tracked as pressure sources.
- **Plug-and-play, zero build dep** — `libXi` is `dlopen`'d at runtime (`libXi.so.6`, present on every X11 desktop) and the XInput2 headers are vendored under `external/xinput2/`, so **`libxi-dev` is not required to build** (only the ubiquitous libX11 dev headers; verified compiling with `libxi-dev` absent). No tablet / no lib / non-Linux → graceful no-op (`pressure()==1.0`), mouse behaviour unchanged.
- **Brush response** — pressure drives brush **strength and size** via independent floors (`PRESSURE_STR_FLOOR 0.05`, `PRESSURE_SIZE_FLOOR 0.40`, `PRESSURE_GAMMA` in `main.cpp`): a light touch fades strength to near-nothing while keeping a usable footprint; hard press digs in. xf86-input-wacom's pressure curve is applied driver-side, so the user's Krita/xsetwacom tuning carries over. Wired through `DabContext::eff_brush_size` (per-dab) + an `eff_strength` multiplier — applies to all brushes. **K** toggles pressure on/off (default on); on-screen notification on tablet detect/connect. GUI-verified with a Wacom One by Wacom M.

## 2026-06-01 — SDF voxel-merge (post-v0.1.0)

- **Voxel-merge ("join for print")** — new `src/sdf.cpp` / `include/sdf.h` / `include/mc_tables.h`: join a selection of mesh entities into one watertight manifold via the GPU-native robust voxel-remesh recipe (pooled triangle soup → unsigned distance band → generalized-winding-number sign → marching cubes), no BVH, no CPU math path. Three compute dispatches over flat SSBOs (bindings 21–27 in `compute.h`): (1) per-triangle distance splat, `atomicMin` on float bits + Ericson closest-point, degenerate-tri guard; (2) winding-number sign — evaluated **only at band corners**, with a CPU multi-source flood fill propagating the inside/outside bit into the off-band bulk (the brute-force O(corners·tris) pass is infeasible on weak GPUs, and MC only interpolates band-to-band edges, so off-band corners just need a consistent sign — spec gotcha 3); (3) marching cubes (count pass → exact alloc → write pass, `atomicAdd` slots, winding oriented to the field gradient). Single readback → hash-weld to an indexed `Mesh` → `Scene::merge_selected_into` (consumes the selection, preserves unselected entities, drops the inputs' multires). MC triangle table validated offline (256/256 cases edge-consistent).
- **UI** — "Merge" button in the ops island + **J** shortcut → Y/N confirm dialog with **[ / ]** resolution nudge (64–256, default 128) → progress overlay → `watertight: N comp, B bnd, M nonmf` report in the result notification.
- **Binary STL export** — `Mesh::export_stl` (per-tri facet normal + 3 verts, 80-byte header, uint32 count; `scale` arg for real-world/mm sizing); export dialog filter is now `.obj,.stl` with extension dispatch.
- Verified at the GUI on a fresh icosphere (watertight, 1 component, 0 boundary edges). **Known limitations (deferred):** sculpted/edited meshes don't yet merge cleanly, and MC output topology is irregular — both to be addressed next. Keep-largest-component toggle and the mm-scale UI field also deferred (`export_stl` already takes the scale arg; the union-find in `remesh.cpp:1283` is the intended source for keep-largest).
